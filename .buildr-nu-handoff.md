## What buildr-nu Is

A local-first development platform that combines three pillars into one UI.
Runs on localhost:3000. No auth, no cloud, no database — filesystem + localStorage.

### Pillar 1: BUILD (Buildr)
- **Free Tools**: Stack Deriver, Module Decomposer, Design Deriver, Constraint Builder, Wave Planner, QA Gate Builder
- **Workspace Generator**: Chat onboarding → derive decisions → generate complete workspace to disk
- Uses Forge engine concepts. Generates WORKSPACE.md, PROJECT.md, agents/, waves/, etc.

### Pillar 2: UNDERSTAND (RepoBrain)
- **Codebase Intelligence**: Connect local repo → file tree, code viewer with syntax highlighting
- **Search**: Semantic + lexical code search across files
- **Q&A with Citations**: Ask questions about a codebase, get grounded answers with file:line refs
- Existing codebase: 190 files, 35K LOC at C:\Users\robin\Pictures\repobrain\src (and rescue copy at C:\Users\robin\Downloads\buildrhel\repobrain-rescue)

### Pillar 3: MANAGE (Toolkit)

- **Vault Browser**: Reads buildrhel/vault/, tree navigation, view content
- **Agent Registry**: View agent hierarchies, routing patterns, model selection, coordination
- **MCP Monitor**: Active MCP servers, their tools, health status

### Module 11: Autonomous Memory System (from rm3mbr)
Cross-session, cross-model, cross-project memory.

**Source code**: C:\Users\robin\Downloads\dr3mr\rm3mbr\
**Key files**:
- memory_manager.py — Core engine (data structures, store, bootstrap generation)
- memory_integration.py — Drop-in setup (auto-detection, project templates, CLI)
- AGENT_PROTOCOL.md — How agents interact with memory autonomously

**Architecture**:
```
.memory/
├── BOOTSTRAP.md          ← Cold-start: agent reads ONLY this
├── manifest.json         ← Machine-readable index
├── sessions/             ← Isolated per session-ID
├── discoveries/          ← Context-directed insights
├── goals/                ← Active + completed goals with bidirectional refs
├── accomplishments/      ← Completed goals with full traceability
├── skills/               ← Extracted reusable methods/patterns
└── phases/               ← Phase summaries with cross-references
```

**Key capabilities**:
- `auto_memory(project_name)` — works in ANY project
- `memory.bootstrap()` — cold-start context in one file read
- `memory.note()` — directional discoveries (planning/executing/reviewing/debugging)
- `memory.set_goal()` — goals with depends_on, enables_existing, opens_doors
- `memory.finish_goal()` — extracts reusable skills automatically
- 5-phase progression: crawl → creep → walk → run → fly
- Session isolation with unique IDs — multi-agent safe
- Bidirectional references — every entry links forward and backward

**Why this matters for buildr-nu**:
- Replaces Buildr's bash-script discovery system with a proper Python API
- Model-agnostic format (markdown + JSON) — Gemini, Codex, Claude all read it
- BOOTSTRAP.md = one-file cold-start for any agent in any model
- Skill extraction = the "fast path" — skip phase history, use the capability

**Integration plan**:
1. Embed memory_manager.py + memory_integration.py into buildr-nu
2. Expose in UI: /memory page with goals, discoveries, skills, phases
3. Every tool/intelligence scan writes discoveries via this API
4. Collab-space view: who's working on what across sessions and models

### Module 12: Collab Space
Shared workspace for multiple LLMs and sessions.

**Concept**:
```
shared-memory/
├── entities.jsonl     ← Facts about the project (persistent)
├── decisions.jsonl    ← Decisions + rationale (append-only)
├── active-context.md  ← What's happening right now
└── agent-log.jsonl    ← Who did what, which model, when
```

Every agent (Claude, Gemini, Codex) reads active-context.md at start,
writes to entities/decisions during work, logs to agent-log at end.
Format is model-agnostic — pure markdown + JSON.

**Page**: /collab with live view: who's working, latest decisions, timeline

```bash

# Intelligence loop — works anywhere with a vault/
```

### Module 15: securIT — System Security Agent

Not just a module — a system-level agent like operator/executor/smith/scout/rescue.

**Three layers:**

**Layer 1: STRUCTURAL (prevents by design)**
- Path allowlists: agents can ONLY access workspace directory + explicitly approved paths
- Secret scanner: all generated files checked for API key patterns before write
- Canary/honeypot files in sensitive dirs (~/.ssh/, email stores, credential dirs)
- MCP communication tools (Slack, email) require per-invocation confirmation
- No agent gets Write + Bash unrestricted — tool permissions per role

**Layer 2: MONITORING (detects during execution)**
- agent-log.jsonl tracks every tool call, every file access, every MCP invocation
- Anomaly detection: agent accessing files outside its declared scope → alert
- Canary trip detection: .canary-DO-NOT-READ files trigger immediate alert
- Rate monitoring: unusual volume of file reads/writes/API calls → flag
- Cross-reference: agent declared intent vs actual file access pattern

**Layer 3: RESPONSE (acts on detection)**
- Canary tripped → pause agent, log full context, alert user
- Secret detected in output → redact before write, log incident
- Scope violation → block action, log, continue with reduced permissions
- Post-session audit: review agent-log for patterns that suggest drift

**Skills this agent needs:**
- `vault/skills/secret-scanning.md` — detect API keys, tokens, passwords in text
- `vault/skills/path-allowlist.md` — configure and enforce file access boundaries
- `vault/skills/canary-placement.md` — where and how to place honeypot files
- `vault/skills/agent-audit.md` — review agent-log for suspicious patterns
- `vault/constraints/no-unrestricted-tools.md` — never give full Bash+Write without scope

**Hooks integration (settings.json):**
```json
{
  "PreToolUse": [
    { "matcher": "Bash", "hooks": [{ "type": "command", "command": "check for dangerous patterns" }] },
    { "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "check path is in allowlist" }] },
    { "matcher": "WebFetch", "hooks": [{ "type": "command", "command": "check URL is not internal/private" }] }
  ],
  "PostToolUse": [
    { "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "scan output for secrets" }] }
  ]
}
```

**Portability:** Like all infra modules, securIT works in any project.
Copy the hooks config + canary files + audit script to any repo.
