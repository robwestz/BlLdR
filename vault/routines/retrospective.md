# Routine: Post-Phase Retrospective

## When to Run
After completing each wave/phase of the project.

## Checklist
- [ ] What went well in this phase? (Name one specific thing)
- [ ] What went poorly? (Name one specific thing)
- [ ] What should we do differently next phase? (One actionable change)
- [ ] Did we stay within LOC budget for this phase?
- [ ] Were the vault items selected for this phase useful?
- [ ] Should any vault item be added based on what we learned?

## If Any Check Fails
Note the failing item in state/orchestration.yaml under `decisions`.
If a new vault item should be created, add it to `backlog` in state.

## Duration
2 minutes.
