# Skill: Systematic Topic Research

## When to Use
Before making implementation decisions that depend on external information,
best practices, or options you haven't directly verified.

## Prerequisites
- A specific question to answer (not a broad topic to explore)
- Access to web search or documentation

## Steps
1. State the specific question in one sentence before searching anything
2. Search with the exact technical terms — avoid paraphrasing
3. Collect 3-5 sources; stop collecting when sources start repeating each other
4. For each source: identify the claim, the evidence, and the recency
5. Cross-check contradicting claims — if two credible sources disagree, note both
6. Extract only what changes a decision or prevents a mistake
7. Write a 3-5 bullet synthesis: what is true, what to avoid, what is unclear
8. Document where each key finding came from (source + date)

## Verification
- [ ] The original question is answered explicitly, not implied
- [ ] At least two independent sources support each key finding
- [ ] Claims older than 2 years are flagged as potentially stale
- [ ] Contradictions are named, not silently resolved
- [ ] Synthesis is actionable: every bullet tells the agent what to do or avoid

## Common Mistakes
- Searching broad topics then drilling down: Start specific, expand only if stuck
- Accepting the first result: Check at least three; quality varies wildly
- Treating documentation as equivalent to real-world experience: Note the difference
- Reporting "it depends" without specifying what it depends on: Always name the conditions
