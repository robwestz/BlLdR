# Skill: Internationalization

## When to Use
Building any project supporting multiple languages.

## Steps
1. Create a translations directory: `/locales/[lang]/common.json`
2. Store all user-visible strings as key-value pairs in translation files
3. Implement a language context/provider that loads the active language
4. Create a `t(key)` function that resolves a key to the current language string
5. Default to the primary language if a translation key is missing
6. Add a language switcher component (dropdown or flag buttons)
7. Store the selected language in localStorage for persistence
8. Reflect the language in the URL path or query parameter

## Verification
- [ ] All user-visible strings come from translation files, not hardcoded
- [ ] Language switcher changes all text on the page
- [ ] Missing translation keys fall back to primary language
- [ ] Selected language persists across page reload

## Common Mistakes
- Hardcoded strings "just for now": Extract from day one. Retrofitting is 5x harder.
- Translating component names: Only translate user-visible text, not code identifiers.
- Forgetting date/number formatting: Locale affects more than just strings.
