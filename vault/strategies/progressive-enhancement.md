# Strategy: Progressive Enhancement

## When to Apply
Building any feature that could be simple or complex.

## The Approach

### Step 1: Build the simplest working version
What is the minimum that delivers value? Build that. Ship it.
A booking form with 3 fields beats a 20-field wizard that isn't done.

### Step 2: Verify it works
Run QA on the simple version. Confirm it's solid.

### Step 3: Layer complexity
If [budget remains and value is clear] → add the next enhancement
If [budget is tight] → the simple version IS the feature
If [unsure if enhancement is needed] → ship without it, observe

## Traps to Avoid
- Building the complex version first: Always start simple, add layers
- "It's not ready without feature X": If the simple version works, it's ready
